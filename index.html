5 Honestly Stated Limitations — §6
1
Synthetic Layer 2 data
354 scripted windows ≠ production diversity. Highest-priority fix: retrain on real logs within 60 days of launch.
2
Layer 1 in log-only mode
S_L1 = 0 during campaign. Cross-layer formula implemented but not empirically validated under dual-layer conditions.
3
Polling latency
10s is polling, not algorithm. Streaming = sub-second.
4
Adversarial robustness — most critical gap
No test against threshold-aware attacker (low-rate, UA-spoofed, distributed).
5
Graph-based detection
Coordinated fraud rings need GNN [12] — requires mature transaction graph from production.

TreeSHAP — Equation 3
f(x) = φ₀ + Σⱼ₌₁ᵈ φⱼ(x)
φⱼ = Shapley value for feature j → how much it pushed the prediction
φ₀ = base rate (expected output over training data)
Exact computation O(TLD²) — T=trees, L=leaves, D=depth
Additive: Σφⱼ = f(x) − φ₀ — each feature's contribution is accountable
Why SHAP matters operationally
EcomFraudEX [13] user study: analysts significantly more likely to act on SHAP-explained alerts than on unexplained probability scores. Explainability is not cosmetic — it changes operational effectiveness.

Backup — Correlation Engine Deep Dive
Backup · Q&A14 / 18
S_final(i, w) = 0.6 · S_L2(i, w) + 0.4 · S_L1(i, w)
Why 0.6 for L2?
Application-layer signals are more specific to the SoukDZ threat model. A high S_L2 is unlikely to be generated by legitimate traffic at attack rates — failed_logins, scan_score, new_accounts are highly discriminative.
Why 0.4 for L1?
Network signals are strong for DDoS and PortScan but weak for S2 (credential stuffing) and S5 (fake accounts) — distributed patterns produce modest network anomalies. Over-weighting L1 would cause misses.
Graceful degradation
When Layer 1 unavailable (log-only mode): set S_L1 = 0 → S_final = 0.6 · S_L2. System continues functioning at reduced confidence. This is what happened during the Kali campaign (S_final ≈ 0.354–0.360, MOYEN).

Backup — Why MOYEN and Not CRITIQUE?
Backup · Q&A15 / 18
S_final = 0.354–0.360 → MOYEN — not CRITIQUE. Why?
L1 was not running
Scapy capture not active during campaign → S_L1 = 0 for all alerts. So S_final = 0.6 · S_L2 only. A fully dual-layer run would yield: S_final = 0.6·0.6 + 0.4·0.9 ≈ 0.72 (ÉLEVÉ) minimum.
Hydra terminates after success
Hydra's 12-attempt brute force campaign (3×4 passwords) terminates after finding the credential. failed_logins never reaches 10 in a single 5-min window → ua_anomaly fires, not the Brute Force heuristic.
Nikto distributes 404s
8,101 Nikto requests scan many different URI patterns → 404s spread across them. No single URI accumulates scan_score > 5 in one window. In production with sustained scanning, the threshold would be crossed.
In a production deployment processing thousands of attempts, both failed_logins > 10 and simultaneous L1 network signals would push S_final to CRITIQUE (>0.85) routinely. The Kali campaign used a small, controlled dataset — it is a proof of concept, not a worst-case simulation.
